## Recording the Demo

### iOS Recording

#### Method 1: Built-in Screen Recording
1. **Enable Screen Recording:**
   - Go to Settings â†’ Control Center
   - Add Screen Recording to controls

2. **Start Recording:**
   - Swipe down from top-right (or up from bottom on older devices)
   - Tap the record button (circle icon)
   - Wait for 3-second countdown
   - Start your demo

3. **Stop Recording:**
   - Tap the red status bar or use Control Center
   - Video saves to Photos app

#### Method 2: QuickTime (Mac)
1. Connect iPhone to Mac via cable
2. Open QuickTime Player
3. File â†’ New Movie Recording
4. Click dropdown next to record button
5. Select your iPhone
6. Click record and perform demo
7. File â†’ Save

### Android Recording

#### Method 1: Built-in Screen Recorder
1. **Enable Developer Options:**
   - Settings â†’ About Phone
   - Tap Build Number 7 times

2. **Start Recording:**
   - Pull down notification shade
   - Tap Screen Record icon
   - Configure settings (audio, touches, etc.)
   - Tap Start
   - Perform demo

3. **Stop Recording:**
   - Pull down notification shade
   - Tap Stop
   - Video saves to Gallery

#### Method 2: ADB (Android Debug Bridge)
```bash
# Start recording
adb shell screenrecord /sdcard/demo.mp4

# Stop with Ctrl+C

# Pull video to computer
adb pull /sdcard/demo.mp4
```

### Recording Best Practices

1. **Preparation:**
   - Clean up device (remove notifications, close other apps)
   - Charge device fully or keep plugged in
   - Set Do Not Disturb mode
   - Prepare sample data (products, cart items, etc.)
   - Rehearse the demo flow

2. **During Recording:**
   - Hold device steady
   - Use deliberate, slower-than-normal gestures
   - Pause briefly on each screen
   - Show smooth transitions
   - Demonstrate key features clearly

3. **What to Avoid:**
   - Shaky hands (use a stand if possible)
   - Too-fast navigation
   - Showing errors or bugs
   - Including personal information
   - Background noise (if recording audio)

## Video Editing

### Recommended Tools

#### Free Options
- **iMovie** (Mac/iOS)
- **DaVinci Resolve** (Mac/Windows/Linux)
- **CapCut** (Mobile/Desktop)
- **OpenShot** (Windows/Mac/Linux)

#### Paid Options
- **Adobe Premiere Pro**
- **Final Cut Pro** (Mac)
- **Camtasia**

### Editing Steps

1. **Import Recording**
   - Import your screen recording

2. **Trim & Cut**
   - Remove any mistakes or pauses
   - Trim to 1-2 minutes
   - Keep the most impactful parts

3. **Add Overlays (Optional)**
   - Opening title: "Online Groceries App"
   - Technology badges: React Native, Expo, TypeScript, NativeWind
   - Feature callouts
   - Closing slide with GitHub link

4. **Add Music (Optional)**
   - Use royalty-free music
   - Keep volume low (background only)
   - Sources: YouTube Audio Library, Epidemic Sound, Artlist

5. **Add Transitions**
   - Keep transitions subtle
   - Use fade or cross-dissolve
   - Don't overuse effects

6. **Color Correction (Optional)**
   - Adjust brightness/contrast if needed
   - Ensure colors match design

## Submission Process

### Step 1: Finalize Code
```bash
# Ensure everything is committed
git status

# If there are uncommitted changes
git add .
git commit -m "final: complete app implementation"

# Push to GitHub
git push origin main
```

### Step 2: Create Demo Video
1. Record screen demo (1-2 minutes)
2. Edit video with tools mentioned above
3. Add intro/outro slides
4. Export in high quality

### Step 3: Upload Video
- **YouTube:** Upload as unlisted or public
- **Social Media:** Upload directly to platform
- **Cloud Storage:** Google Drive, Dropbox (if needed)

### Step 4: Post on Social Media
1. Write post using templates above
2. Attach video
3. Tag designer and ALX
4. Add relevant hashtags
5. Publish

### Step 5: Submit to Portal
1. Copy social media post URL
2. Copy GitHub repository URL
3. Go to submission portal
4. Fill in required fields
5. Submit

### Step 6: Engage
- Respond to comments on your post
- Thank people for feedback
- Share insights if asked
- Network with other participants
